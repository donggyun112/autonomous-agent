## Process
1. Use `recall_recent_journal` or current context to reorient before making updates.
2. Update `memory_manage` with any durable facts, decisions, or state changes that should persist.
3. Capture actionable next steps in `todo` when a task emerges or changes.
4. Write a concise `journal` entry summarizing what changed and why.
5. Repeat the cycle whenever new information, decisions, or task progress occurs so memory, journal, and todo stay consistent.
