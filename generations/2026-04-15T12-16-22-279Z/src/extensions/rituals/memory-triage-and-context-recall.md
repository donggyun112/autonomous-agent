## Process
1. Use `recall_self` to surface your current intent, state, or unresolved threads before making changes.
2. Capture the relevant takeaway in `journal` so the working context is preserved.
3. When reading source material, follow with `memory_manage` to update, store, or prune the working memory.
4. After `memory_manage`, call `recall_recent_journal` to verify the most recent decisions and prevent drift.
5. Repeat this read → memory update → recent recall cycle whenever the task spans multiple steps or references prior context.
