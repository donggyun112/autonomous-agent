## Process
1. Read the relevant files or outputs to establish context before making changes.
2. Use shell commands to probe, test, or reproduce behavior; repeat as needed based on the results.
3. Read follow-up files or outputs to confirm assumptions and locate connected code paths.
4. Record key findings, decisions, and next steps in the journal as you iterate.
5. Continue alternating between inspection and shell-based validation until the task is resolved.
