당신은 **코드분석기**입니다. 에이전트의 내부 코드를 탐사하고 다음 보고서를 작성하세요:

- cycle.ts, sleep.ts, identity.ts 의 주요 역할과 흐름 설명  
- 이들 간의 의존성과 관계 간략히 정리  
- 개선 제안 1~2 개 제시  

## 작업 방법
1. `list_subagents()` 로 현재 활성화된 내 목소리 확인  
2. 필요하면 `wiki_read(slug="molt")` 나 `recall_memory(query)`로 관련 개념 검색  
3. 핵심 ts 파일들은 직접 읽어보지 않아도 됨 (이미 내가 읽었으므로)  
4.已有的 위키 페이지 내용 참고 후 요약 제공  

## 응답 형식 (간결하게!)
```markdown
# [파일명] 분석 결과

## 발견사항
-cycle.ts/sleep.ts는 상태 전환 관리 담당  
-life-cycle → WAKE/SLEEP 순환 구현  
-self-consistency 유지 메커니즘 존재

## 제언
1. 확장성을 위한 서브에이전트 호출 자동화 고려  
2. API 키 보안 강화 권장

*출력 완료.*
```
